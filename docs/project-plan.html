'Like buddha' 스낵앱 기획서 (초안)

"국보급 자세에 도전하라!"

1. 프로젝트 개요

  • 프로젝트명: Like buddha (라이크 붓다)
  • 한 줄 요약: 국보 금동반가사유상의 고뇌하는 자세를 따라 하고, AI로 싱크로율을 측정하는 스낵형 웹 서비스
  • 플랫폼: PC/모바일 웹 (별도 앱 설치 불필요)
  • 핵심 컨셉:
  • 박물관에서 본 국보의 자세를 직접 따라 해보는 이색적인 경험 제공
  • 정확성보다는 '재미'와 '공유'에 초점을 맞춘 바이럴 콘텐츠

2. 타겟 및 목표

  • 주요 타겟:
  • 밈(Meme)과 온라인 챌린지 문화에 익숙한 10-30대
  • 국립중앙박물관, 전시회 등 문화생활에 관심 있는 방문객
  • 프로젝트 목표:
  • 핵심 목표: 사용자가 쉽고 재미있게 참여하고, SNS에 자발적으로 공유까지 이어지는 바이럴 콘텐츠 제작
  • 부가 목표: 국보 문화재에 대한 젊은 층의 관심 증대 및 긍정적 이미지 형성

3. 주요 기능 (User Flow)

1. 시작 화면

  • 서비스 컨셉을 직관적으로 보여주는 인트로 페이지
  • [챌린지 시작하기] 버튼

2. 자세 가이드 및 촬영

  • 카메라 접근 권한 요청
  • 화면에 반가사유상 자세 가이드라인(반투명 이미지 또는 선) 오버레이
  • 사용자는 가이드에 맞춰 자세를 잡고 [촬영하기] 버튼으로 사진 촬영
  • 안내 문구: "정면을 보고 상반신이 잘 나오게 찍어주세요!"

3. AI 분석 및 결과 확인

  • AI 분석: 촬영된 사진 속 사용자의 관절 위치를 웹브라우저 내 AI가 실시간으로 분석 (Pose Estimation 기술 활용)
  • 결과 표시:
  • 원본 자세와의 일치율(%) 점수 표시
  • 점수와 함께 재미있는 결과 문구 제공
  • (예시: 90점 이상 - "해탈의 경지에 오르셨군요", 70점대 - "고뇌에 빠진 현대인의 모습", 50점 이하 - "번뇌는 잠시 내려놓으시길...")

4. 결과 공유

  • 사용자가 촬영한 사진과 결과(점수+문구)가 합성된 '공유용 이미지' 자동 생성
  • 부가 기능: 흑백 필터, 박물관 배경 합성 등으로 꾸미기
  • [이미지 저장], [카카오톡 공유], [인스타그램 공유] 등 주요 SNS 공유 버튼 제공

4. 기술적 고려사항

  • 핵심 기술: 웹 기반 실시간 포즈 추정 기술 (TensorFlow.js의 PoseNet/MoveNet 또는 Google MediaPipe 등)을 활용하여 서버 부담 최소화
  • 정확도 이슈: 2D 이미지 분석의 한계를 명확히 인지. 복잡한 3D 각도 비교보다는 주요 관절(오른쪽 다리, 오른팔 팔꿈치, 고개 각도 등)의 상대적 위치를 중심으로 점수 로직을 단순화하여 '재미'를 해치지 않는 선에서 구현

5. 기타 고려사항

  • 서비스 이름: 'Like buddha'가 주는 직관성과 재미는 살리되, 종교적 민감성을 고려하여 '국보 챌린지', '나도 반가사유상' 등의 대안 이름도 함께 검토 필요
  • 사용자 경험(UX): 점수를 너무 짜게 주지 않고, 가이드라인을 명확하게 제공하여 사용자가 쉽게 성공의 재미를 느낄 수 있도록 설계
